# Understanding ADHD

> Context for building ADHD-friendly web experiences

## What is ADHD?

A neurological condition that affects executive functions. **It's NOT a lack of willpower.**

## Executive Functions Affected

| Function | How It Affects |
|----------|----------------|
| **Activation** | Can't start tasks even when they want to |
| **Focus** | Easy distraction OR extreme hyperfocus |
| **Effort** | Rapid mental fatigue |
| **Emotion** | Intense reactions, rejection sensitivity |
| **Working Memory** | Forget what they were about to do |
| **Time Management** | "Time blindness" |

## Time Blindness (Critical Concept)

- 5 minutes can feel like 1 hour (if boring)
- 3 hours can feel like 10 minutes (in hyperfocus)
- Deadlines don't feel "real" until they're imminent

**-> This is why VISUAL timers are critical in the app**

## The Cycle We Must Break

```
1. Find new app (enthusiasm)
        |
2. Set everything up (energy)
        |
3. System too rigid
        |
4. Don't meet planned goals
        |
5. Feel like a failure
        |
6. Abandon the app
        |
7. Search for another app -> repeat
```

**Our solution:** Break this cycle with flexibility and celebration of imperfect progress.

## Website Implications

The website is the first touchpoint for potential users. It must:

| Challenge | Our Web Approach |
|-----------|-----------------|
| Sensory sensitivity | Calm colors, subtle animations, no flashing |
| Decision fatigue | Clear CTA, minimal choices per section |
| Short attention span | Concise copy, visual hierarchy, scannable |
| Shame from failure | Positive, empathetic messaging throughout |
| Distraction | Clean layout, focused content, no clutter |
| Trust issues with apps | Social proof, transparent pricing, honest tone |

## What Website Visitors Need

### Do
- Show the app's value in 5 seconds (hero section)
- Use calm, welcoming visuals
- Make the waitlist/download action obvious
- Keep copy short and scannable
- Show understanding of ADHD challenges

### Don't
- Use aggressive sales language
- Create urgency or pressure
- Overwhelm with features list
- Use clinical/medical terminology
- Create anxiety with design choices

## Real User Complaints (From Competitor Reviews)

> "Learning curve steep... endless customizations take too much time"

> "Logs me out randomly... interruptions devastating for ADHD"

> "Before I'm done with first item, it's at fourth - feel defeated"

> "Entire schedule disappeared after refresh"

## Our Core Response

**"Forgiving by Design"** -- An app that forgives, adapts, and celebrates imperfect progress.

| What Others Do | What We Do |
|----------------|------------|
| "You failed 2 tasks" | "You completed 3 of 5" |
| Rigid time slots | Auto-adjusting schedules |
| Same demands always | "Difficult day" mode |
| Complex setup | 2-minute onboarding |
